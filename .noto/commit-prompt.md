# Commit Message Guidelines

## Format
Use a simple `type: description` format.

## Style Rules
- **Tense**: Imperative/present tense (e.g., "add", "set up", "init")
- **Capitalization**: Lowercase first letter of the description
- **Length**: Concise and to the point
- **Tone**: Technical and direct

## Commit Types
- `docs`: For documentation updates or additions
- `feat`: For new features or capabilities
- `chore`: For routine tasks, tooling changes, or repository setup

## Scope Usage
No scopes are used in the current commit history.

## Description Patterns
Start the description with an active verb that describes the change. Focus on what the commit does.

## Examples from History
- docs: add MIT license
- feat: add Ballerina WASM runtime and NodeFS adapter
- chore: set up Turborepo monorepo and biome tooling
- chore: init repo
